## 第二部分：项目概述

### 2.1 项目描述

- **项目名称**: 智能个人财务记账助手
- **目标用户**: 大学生、职场新人
- **核心问题**: 日常消费记账繁琐易遗漏,对收支结构不清晰,缺乏理财规划意识
- **解决方案**: 通过自然语言交互完成记账,结合长期记忆功能沉淀用户消费习惯,提供智能化的理财建议和消费分析

### 2.2 用户画像

**用户画像1: 大学生小王**

| 属性 | 描述 |
|------|------|
| 基本信息 | 20岁,大三学生,每月生活费1500元 |
| 核心需求 | 了解每月钱花在哪里,控制不必要的消费 |
| 痛点 | 记账太麻烦,经常忘记,不知道哪些消费可以节省 |
| 使用场景 | 随时随地记录消费,月底查看消费分析 |

**用户画像2: 职场新人小李**

| 属性 | 描述 |
|------|------|
| 基本信息 | 24岁,入职半年,月薪6000元 |
| 核心需求 | 开始有储蓄计划,学习理财知识,优化收支结构 |
| 痛点 | 工作忙碌没时间记账,不知道如何分配收入,缺乏理财规划 |
| 使用场景 | 下班后快速记账,周末查看财务报告,制定下月预算 |

**用户画像3: 理财新手小张**

| 属性 | 描述 |

| 核心需求 | 建立理财习惯,学习投资知识,实现财务自由 |
| 痛点 | 对理财知识一知半解,不知道从何开始,容易冲动消费 |
| 使用场景 | 记录每笔消费,获取理财建议,跟踪储蓄目标 |

### 2.3 核心场景

**场景1: 快速记账**

| 用户输入 | 系统响应 | 涉及AI技术 |
|---------|---------|-----------|

| "给女朋友买了束花,99元" | 已记录！礼物 99元(浪漫消费💝) | 情感分析,个性化标签 |

| 用户输入 | 系统响应 | 涉及AI技术 |

| "我在哪里花得最多" | 分析结果:餐饮类消费最多,共711元<br>建议:可以适当减少外卖次数 | 消费分析,智能建议 |

### 2.4 智能特性

本项目的AI-Native特性包括:

- **无需注册登录**: 采用设备身份识别,首次访问自动生成设备ID,存储在浏览器localStorage,后续访问自动识别

- **自然语言交互**: 用户通过自然语言记账和查询,系统理解用户意图并自动提取关键信息(金额、类别、时间等)

- **长期记忆(核心特性)**:
  - 记住用户的消费习惯(如:常去的餐厅、喜欢的消费类别、平均消费水平)
  - 记住用户的财务目标(如:储蓄目标、购买计划)
  - 无需重复说明个人情况,系统自动使用记忆提供个性化服务

- **主动服务(核心特性)**:
  - 定时推送消费提醒(如:今天还没记账哦~)
  - 月末主动生成财务报告
  - 智能提醒消费异常(如:本月餐饮消费比上月增长50%)
  - 储蓄目标进度提醒

- **情感交互**: 识别用户的情绪状态,提供人性化回复(如:消费过高时委婉提醒,达成目标时给予鼓励)

- **智能分析**: 基于用户历史数据,预测未来消费趋势,提供优化建议

## 第三部分：核心功能

### 3.1 功能架构图

```
智能个人财务记账助手
├── 智能对话层
│   ├── 自然语言理解(NLU)
│   ├── 意图识别
│   └── 信息提取
├── 核心业务层
│   ├── 记账功能
│   ├── 消费分析
│   ├── 理财规划
│   └── 数据统计
├── 智能服务层
│   ├── 长期记忆
│   ├── 主动提醒
│   └── 智能推荐
└── 数据存储层
    ├── 用户数据(消费记录、目标设置)
    ├── 消费知识库(消费类别、理财建议)
    └── 记忆存储(用户习惯、偏好)
```

### 3.2 核心功能

**功能1: 智能记账**

- 通过自然语言完成记账,支持语音和文字输入
- 自动识别消费类别、金额、时间
- 支持批量记账和定时记账
- 提供智能分类建议

**功能2: 消费分析**

- 自动生成消费报告(日报、周报、月报)
- 可视化展示消费结构(饼图、柱状图、趋势图)
- 消费趋势分析和预测
- 同比、环比对比分析

**功能3: 理财规划**

- 帮助设定储蓄目标和消费预算
- 根据消费习惯提供个性化理财建议
- 智能推荐理财产品和投资组合
- 目标进度跟踪和提醒

## 附录

### A1. 技术栈选择

| 层次 | 技术选择 | 说明 |
|------|---------|------|
| 前端框架 | Vue 3 + Vite | 现代化前端框架,开发效率高 |
| 后端框架 | Node.js + Express | 轻量级,易于部署 |
| AI能力 | Claude API / OpenAI API | 提供自然语言理解和生成能力 |
| 对话框架 | LangChain | 简化LLM应用开发 |
| 数据存储 | SQLite | 轻量级数据库,适合本地存储 |
| 数据可视化 | ECharts | 丰富的图表类型,交互性好 |
| 部署方案 | Vercel / Netlify | 免费托管,自动部署 |

### A2. 风险评估与应对

| 风险 | 风险等级 | 应对方案 |
|------|---------|---------|
| API调用费用高 | 中 | 使用免费额度,或考虑使用本地开源模型 |
| 自然语言理解准确率不足 | 高 | 增加示例训练,使用Few-shot Prompting |
| 数据隐私安全 | 高 | 采用本地存储,不传输敏感数据,遵守隐私法规 |
| 开发时间不足 | 中 | 分必做和选做功能,优先完成核心功能 |
| 开源项目学习成本高 | 低 | 选择成熟的开源项目,参考文档和示例 |

### A3. 创新点与特色

1. **AI-Native设计理念**: 不同于传统记账软件的表单输入,采用"对话即操作"的自然语言交互方式

2. **长期记忆机制**: 系统会记住用户的消费习惯和偏好,提供个性化服务,无需用户重复说明

3. **主动服务**: 主动推送财务提醒和分析,像真人财务顾问一样关心用户

4. **跨学科融合**: 结合会计专业知识和软件工程技术,提供更专业的理财建议

5. **极简体验**: 无需注册登录,打开即用,降低使用门槛
